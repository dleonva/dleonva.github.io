Guía Completa: SVG y Diseño Vectorial en la Web

Esta guía detalla los conceptos fundamentales detrás de los Gráficos Vectoriales Escalables (SVG: Scalable Vector Graphics), su relación con el lenguaje XML (Extensible Markup Language), cómo integrarlos en proyectos web y un desglose técnico línea por línea de un diseño geométrico.

Para mayor detalle puedes consultar: SVG: Scalable Vector Graphics

1. Conceptos Fundamentales

¿Qué significa SVG?

SVG significa Scalable Vector Graphics (Gráficos Vectoriales Escalables). Es un formato de imagen basado en texto que utiliza el lenguaje XML para describir imágenes bidimensionales mediante fórmulas matemáticas y geometría (líneas, curvas, polígonos, colores) en lugar de una cuadrícula de píxeles estática.

Vectores vs. Píxeles (Mapas de bits):

Formatos tradicionales como JPG o PNG son mapas de bits. Si haces zoom sobre ellos, verás los cuadrados individuales (píxeles) y la imagen perderá calidad o se verá borrosa. Un vector (SVG) es una fórmula matemática. Si lo escalas al tamaño de un edificio, la computadora simplemente recalcula la fórmula, por lo que la imagen siempre será 100% nítida, sin importar su tamaño.

¿Qué es XML y su diferencia con HTML?

XML (eXtensible Markup Language - Lenguaje de Marcado Extensible) es un lenguaje de marcado muy estricto diseñado para almacenar y transportar datos, enfocado en qué es la información.

Característica HTML XML (Usado en SVG)
Propósito principal Mostrar datos en pantalla (Diseño y estructura visual). Describir y almacenar datos (Estructura lógica y matemática).
Etiquetas Predefinidas (ej: <p>, <h1>, <img>). Extensibles. En el caso de SVG, usan etiquetas geométricas (ej: <rect>, <circle>,<line> ).
Reglas de sintaxis Flexible y permisivo. A veces el navegador corrige errores si olvidas cerrar una etiqueta. Extremadamente estricto. Si una etiqueta no se cierra, el gráfico no se renderizará o dará error.

2. Cómo insertar un SVG en un documento HTML

Existen varias formas de integrar un gráfico SVG dentro de tu página web, dependiendo de lo que necesites lograr.

A. Código en línea (Inline SVG) - Método recomendado para interactividad

Consiste en pegar el código <svg> directamente dentro del HTML (es el método que estamos usando en esta página). Su gran ventaja es que permite modificar las partes del gráfico mediante CSS (por ejemplo, cambiar el color al pasar el ratón) o manipularlo con JavaScript.

<body>
    <h1>Mi Logo</h1>
    <!-- El código SVG va directamente aquí -->
    <svg viewBox="0 0 100 100">
        <circle cx="50" cy="50" r="40" fill="red" />
    </svg>
</body>

Mi Logo

B. Mediante la etiqueta <img>

Si guardas tu código en un archivo externo llamado logo.svg, puedes llamarlo como cualquier otra imagen. Es útil para mantener el HTML limpio, pero pierdes la capacidad de modificar sus colores internos mediante el CSS de la página principal.

<img src="logo.svg" alt="Logotipo de la empresa" width="250" height="250">

C. Como fondo en CSS (Background Image)

Ideal para patrones decorativos, iconos de interfaz o fondos que se adaptan a la pantalla.

.mi-caja {
    background-image: url('logo.svg');
    background-size: cover;
    background-size: 0.5%;
}

3. El Sistema de Coordenadas SVG

Para entender el código, es vital saber cómo se posicionan los elementos. El plano cartesiano en SVG no funciona igual que en matemáticas tradicionales:

  • El punto de origen (0, 0) está en la esquina superior izquierda del lienzo.
  • El eje X crece hacia la derecha (valores positivos mueven el objeto a la derecha).
  • El eje Y crece hacia abajo (valores positivos mueven el objeto hacia abajo).

4. Análisis del Código: Logo Técnico

4.1. El Lienzo (<svg>)

Configuración del área de dibujo y el sistema de coordenadas principal.

  • xmlns="http://www.w3.org/2000/svg": Define el estándar XML. Indica al navegador que el código contenido debe interpretarse como gráficos vectoriales.
  • viewBox="0 0 250 250": Establece la cuadrícula interna del gráfico. Los dos primeros números (0 0) son las coordenadas de origen. Los dos últimos (250 250) son el ancho y alto relativos internos. Esta es la clave para que el SVG sea escalable.
  • width="250" height="250": El tamaño físico predeterminado en píxeles que ocupará en la pantalla (aunque mediante CSS se puede sobreescribir).

4.2. El Cuadrado Base (<rect>)

Creación de la figura delimitadora con esquinas redondeadas y fondo transparente.

  • x="6" y="6": Coordenadas de inicio. Se desplaza 6px hacia adentro desde la esquina superior izquierda. Esto se hace porque los bordes en SVG crecen desde el centro de la línea matemática hacia afuera; si estuviera en 0,0, la mitad del grosor del borde quedaría recortado y fuera del lienzo.
  • width="238" height="238": Dimensiones de la forma. Es 250 menos los 12px de márgenes totales (6px por la izquierda + 6px por la derecha).
  • rx="44": Aplica un radio de curvatura de 44 píxeles a las esquinas.
  • fill="none": Hace que el interior del rectángulo carezca de relleno (sea totalmente transparente).
  • stroke="#0000FF": Aplica color al trazo (borde). Usa el modelo de color Hexadecimal; `#0000FF` es azul puro (0 rojo, 0 verde, 255 azul).
  • stroke-width="12": Determina el grosor del borde en 12 unidades (píxeles internos).

4.3. Las Líneas de Cruce (<line>)

Dibujo de las guías internas, aplicando patrones de segmentación geométrica.

  • Coordenadas (x1, y1, x2, y2): Definen los puntos de inicio (1) y fin (2) de un segmento recto.
    • Horizontal: Va del pixel X=12 al X=238 manteniendo la altura constante en Y=125 (el centro exacto de 250).
    • Vertical: Va del pixel Y=12 al Y=238 manteniendo la posición horizontal constante en X=125.
  • stroke="#444" stroke-width="2": Aplica un color gris oscuro y un grosor de 2 unidades a la línea trazada.
  • stroke-dasharray="8,6": Crea el efecto de línea punteada/discontinua. Le ordena al motor de renderizado: "Dibuja 8 unidades de trazo, deja 6 unidades de vacío, y repite hasta llegar al punto final (x2, y2)".
  • stroke-dashoffset="3": Ajusta el punto de inicio del patrón discontinuo. Desplaza la secuencia 3 unidades hacia atrás. Esto se calculó matemáticamente para forzar que la parte sólida del trazo pase exactamente por la coordenada central (125), logrando que la intersección sea una cruz perfecta limpia, en lugar de un hueco o una unión desfasada.

5. Código Limpio (Producción)

Este es el código resultante consolidado. Al estar insertado "en línea" (Inline SVG) dentro de este mismo documento HTML, el navegador lo dibuja inmediatamente al leer el código fuente.

<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 250 250" width="250" height="250">
  <rect x="6" y="6" width="238" height="238" rx="44" fill="none" stroke="#0000FF" stroke-width="12" />
  <line x1="12" y1="125" x2="238" y2="125" stroke="#444" stroke-width="2" stroke-dasharray="8,6" stroke-dashoffset="3" />
  <line x1="125" y1="12" x2="125" y2="238" stroke="#444" stroke-width="2" stroke-dasharray="8,6" stroke-dashoffset="3" />
</svg>
