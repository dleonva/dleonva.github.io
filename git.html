Git: Flujo de Trabajo Profesional

Guía de referencia paso a paso para el control de versiones local y sincronización con GitHub desde Visual Studio Code.

1. Modificar (Working Directory)

Realizar y guardar los cambios en los archivos locales.

  • Abre el archivo en Visual Studio Code (ej. .html o .css).
  • Realiza las modificaciones necesarias en el código.
  • Guarda el archivo (Cmd + S).
  • Nota: En la barra lateral izquierda, el ícono de "Control de código fuente" mostrará una notificación azul indicando que Git detectó modificaciones.

2. Preparar los cambios (Staging Area)

Seleccionar explícitamente qué archivos modificados se incluirán en la próxima actualización.

  • Vía VS Code (Interfaz): Ve a la pestaña de Control de código fuente. En la lista "Cambios", pasa el cursor sobre el archivo y haz clic en el ícono (+) para pasarlo a "Cambios almacenados".
  • Vía Terminal: Ejecuta el siguiente comando para agregar todos los archivos modificados:

    git add .


  • Nota: (El punto . significa "agrega todos los archivos modificados").

3. Confirmar (Commit)

Empaquetar los archivos preparados y asignarles un mensaje descriptivo en el historial local.

  • Vía VS Code (Interfaz): En la caja de texto de la pestaña de Control de código fuente, escribe un mensaje descriptivo (ej. "Agrega sección de Git") y haz clic en el botón Commit.
  • Vía Terminal: Ejecuta el comando con tu mensaje entre comillas:

    git commit -m "Mensaje descriptivo del cambio"

4. Enviar a Producción (Push)

Subir el historial de cambios confirmados desde la computadora local al repositorio remoto en GitHub.

  • Vía VS Code (Interfaz): Haz clic en el botón Sync Changes (Sincronizar cambios) que aparece después de hacer el commit.
  • Vía Terminal: Ejecuta el comando para empujar los cambios a la rama principal:

    git push
